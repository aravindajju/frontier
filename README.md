# Frontier

*An Opinionated Method for Forward Deployed Engineering*

Forward Deployed Engineering (FDE) is the practice of embedding product engineers inside customer environments to ship working software against real data, and feeding what they learn back into the product. **Frontier** is an opinionated method for practicing it — defined roles, a time-boxed lifecycle, named artifacts, falsifiable rules, and four metrics — the way Scrum is a method within Agile. The name is the territory: the frontier is where the product meets the unknown, and that is where this work happens.

The role has grown explosively since 2024, but the discipline has no canonical text — no equivalent of the Scrum Guide, XP Explained, or the Google SRE book. This project is that text, written by a practitioner, vendor-neutral, and open.

## The three artifacts

| Artifact | Location | What it is |
|---|---|---|
| Manifesto | `manifesto/` | One page. Values and numbered principles. |
| The Guide | `guide/` | The normative core, 15 pages or fewer. Definitions, roles, lifecycle, artifacts, operating rules, metrics. Versioned with a public changelog. |
| The Book | `book/` | The depth layer. Theory, practices in pattern form, war stories, org design, economics. |

The Guide is the citable canon. The book explains and grounds it; it never contradicts it.
